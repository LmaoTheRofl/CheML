# Архитектура

`chemx parse` создаёт новый `runs/<id>` и записывает manifest. Run output inside `/tmp` запрещён: результаты парсинга должны оставаться в persistent workspace. PyMuPDF извлекает metadata, постраничный текст, layout blocks, таблицы, изображения и page renders. Production Codex-run требует успешный Marker; fallback допускается только для явно непродукционного bundle-building.

Production `chemx parse` и `chemx batch` используют Codex по умолчанию и требуют полный стек PyMuPDF, `pymupdf_layout`, Marker, OCR, MolScribe, RDKit и Codex reviewer. Project-local OCR data и MolScribe runtime/weights обнаруживаются автоматически; environment-команды остаются явными overrides. После bundle-building в workspace копируются только router skill и выбранный domain skill. Gold-маркеры проверяются до и после backend. `CodexBackend` и `OllamaBackend` реализуют один `Prediction` contract; локальный backend направляет `codex exec --oss` через compatibility adapter на `127.0.0.1:11434`, а реальный Ollama слушает `127.0.0.1:11435`. Adapter устраняет конфликт schema grammar с tool calls и восстанавливает имена Gemma function calls, не меняя этапы pipeline. После успешного preprocessing записываются `bundle.json`, `layout.json`, `marker.md/json`, `tables.json`, `ocr.json`, `ocsr.json`, `chemistry_candidates.json` и `tool_manifest.json`. После inference deterministic postprocessor чинит schema shape, канонизирует SMILES через RDKit и пишет `prediction.json`, `prediction.csv`, `schema_diagnostics.json`, `chemistry_diagnostics.json`; reviewer пишет `review.json` и `review_report.md`. После оценки рядом добавляется `reference.csv` с эталонными строками текущей статьи.

`chemx resume runs/<failed-run-id>` продолжает inference и reviewer из готового preprocessing workspace, не повторяя Marker, OCR и OCSR после внешнего сбоя backend.

Evaluator запускается отдельной командой после `inference_complete`. Default gold source — скачанные локально parquet-файлы в `datasets`: evaluator находит parquet рядом с исходным PDF, проверяет имена/порядок/Arrow-типы полей относительно domain contract, извлекает DOI из bundle/PDF и сравнивает строки этой статьи по DOI; если DOI не сопоставился, fallback идёт по `pdf`, затем по нормализованному `title`. Метрика сравнивает мультимножества нормализованных значений независимо для каждого столбца, вычисляет среднее по полям `macro_f1` и записывает `evaluation.json` + `evaluation_metrics.csv`; отфильтрованные эталонные строки дополнительно пишутся в `reference.csv`. `evaluate-batch` агрегирует новейший завершённый run каждой статьи в `article_macro_f1.csv`.

Тяжёлые компоненты разделены на extras: Marker, RDKit, Streamlit и gold loader. Из-за несовместимых ограничений Torch MolScribe запускается subprocess-adapter из отдельного project-local окружения; `CHEMX_MOLSCRIBE_COMMAND` позволяет переопределить автодетект. Docker image скачивает Marker/Surya cache в `/opt/chemx-cache` и MolScribe checkpoint в `/opt/chemx-models`, чтобы runtime не зависел от host-local model files. Docker Compose profile `ollama` добавляет native `ollama` service, one-shot `ollama-pull` для `lukaspetrik/gemma3-tools:27b` и `ollama-adapter`, поэтому `--backend ollama` может работать без host-local Ollama. Базовый проект остаётся пригодным для построения bundle и тестирования без GPU.
